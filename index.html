Across the two sprints we mapped out how every user and organisation will transfer over, began the clear out of empty “ghost” orgs to clean the dataset and ensuring admin reacts automatically when WorkOS data changes. We agreed the sign-in page approach and built the migration tooling that will switch users across when we are ready, capped off with an engineering alignment session to lock in the next steps.
The admin work introduced a new ‘master’ access area inside each user’s profile, executed in a tab style. Within this ‘Access’ tab you can see the user’s Plan, Trial, Devices and Perishable Links all in one place, with safer editing (unsaved-changes warning, cancel button) and automated test coverage to catch regressions. Navigation links and old bookmarks were re-routed to the new pages so nothing breaks for existing admins. Alongside that, multi-org filtering got two targeted fixes: search by Org ID is now live and the results bug in long lists has been fixed.
We started rolling out AI coding agents across our engineering repos by adding AGENTS.md briefing files — a short guide that tells AI coding tools how our team works, what conventions to follow and how to write code that fits our standards. The first wave covered four core I&A repositories: admin-users (the admin tool for managing users and access), bundles (product packages and customer entitlements), sso-proxy (single sign-on between platforms) and users (the core service that stores every account). The aim is consistent, on-brand AI assistance whichever repo an engineer is working in.

Product design

Across the period, product design contributed UX/UI improvements to the Close User Account flow in admin and relocated it into the new ‘Master’ Access page. Alongside delivery work, the team produced research and analysis on the Domains and Bundles pages — mapping initial layout concepts and recommending an approach to migrate Bundles to a new code base with refreshed UI — plus a WorkOS analysis from a product design perspective to inform the upcoming sign-in changes.

02 Multi-org filtering enhancements

Faster, smarter ways for CS and operations to manage users across many organisations.

Search by Org ID + a stubborn pagination bug squashed.

  • Search by Organisation ID - Use the drop down to select whether you want to search by Org name / ID. Type an Org ID into the search bar and find any organisation instantly. You can then switch to searching by org name.
  • Duplicate results bug fixed - long org lists were not fully rendering. Meaning in the scenario of Publicis users would not have been able to see all orgs.

Quicker and more dynamic filtering.

  • CS support workflows - This helps to make CS reporting easier and more streamlined, whilst also providing flexibility.
  • High-volume reliability - Large lists are now stable, which matters for ops teams managing thousands of organisations.
  • Foundation for bulk actions - A clean, reliable list is the prerequisite for the bulk-edit work coming up.

Coming up for multi-org.

  • Sticky header search component - When needing to scroll for long results lists (see attached image).
  • Concept designs - To enable filter by plan in multi-org search.
  • Analysis - Understand feasibility of adding filtering by bundles.
  • Longer term - Explore how we use functionality to improve bulk actions across admin.

05 UX discovery · The path forward

To enable self-serve at scale, we need to fix the foundation - not layer on more workflows.

01 Foundation

Fix the model

Represent customers properly across organisations. One coherent model that mirrors how enterprises actually operate - umbrellas, regions, contracts - with roll-up visibility built in.

  • Org hierarchy & relationships
  • Contract, org,  user mapping
  • Roll-up reporting
02 Governance

Standardise access

A clear, consistent model for permissions and entitlements. Seats, roles, plans, features and datasets - governed in one place, with lifecycle states for trials and tokens.

  • Unified seats, roles, plans
  • Trial & token lifecycle
  • One source of truth
03 Scale

Automate workflows

Remove the human dependency. User setup, seat changes, bundle changes, reporting - automated with guardrails so customers self-serve and support stops being the bottleneck.

  • Self-serve user & seat actions
  • Automated bundle changes
  • Guardrails & audit trail
→ What this enables as we scale

Reduce operational overhead

Move away from manual reporting, setup and access management - free up hundreds of hours across CS, PX and RevOps.

Scale enterprise effectively

Support complex org structures, contracts and access models without increasing cost to serve. Makes onboarding enterprise clients easier.

A trusted, unified system

Align contracts, users and usage in one model. Accurate reporting, better decisions, stronger renewals - internal teams have confidence.
