Feature Focus: Multi-Org Filtering

Feature Focus: Multi-Org Filtering

Giving CS and operations teams faster, smarter ways to manage users across multiple organisations

🔍

Search by Organisation ID

Previously, finding a specific organisation required searching by name — slow if you had an ID but not the exact name
CS teams can now paste an Org ID directly into the search bar for an instant match
Particularly useful for support workflows where clients provide their Org ID in tickets
🐛

Pagination Bug Fix

When scrolling through large organisation lists, the same org was appearing multiple times due to an unstable sort order
Fixed the underlying sort logic so each organisation appears exactly once, regardless of list size
Improves reliability for teams managing high-volume organisation searches
🗺️

What's coming next for Multi-Org

Bulk actions across multiple organisations in a single operation
Filter users by their organisation membership
Export filtered organisation lists for reporting
Organisation hierarchy view for enterprise clients

Design Update: Synthesising our UX Discovery

Design Update: UX Discovery — Proto Personas

We spoke to internal teams across CS, PX, and Engineering to understand how people really use the admin platform. Here's what we found.

Who we spoke to
CS Enterprise Product Experience Engineering Product Design Commercial Operations
👩‍💼
The CS Operator
CS Enterprise / Account Management
Uses admin daily to onboard clients, manage access, and resolve account issues. Needs speed — repetitive tasks slow them down significantly at scale.
Bulk actions Speed Reliability
🛠️
The Power User
Product Experience / Operations
Investigates access issues and permissions problems. Needs clear audit trails and the ability to diagnose exactly what a user can and can't see on the platform.
Audit trail Diagnostics Permissions visibility
🔐
The Security Steward
IT / Security / Compliance
Manages SSO configuration and user lifecycle at an organisational level. Needs confidence that changes are logged, reversible, and won't cause unexpected access issues.
SSO Logging Change management
Key themes emerging from discovery
⚡ Speed over completeness
Users want to complete common tasks in as few clicks as possible — not navigate complex menus
🔎 Visibility of permissions
"What can this user actually see?" is the most common question — and currently very hard to answer quickly
📋 Trust in the system
Users want confidence that changes are logged and reversible — fear of breaking something holds them back
